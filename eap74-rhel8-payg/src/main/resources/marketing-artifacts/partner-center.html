JBoss EAP is an industry-leading Java runtime powering some of the most mission-critical enterprise applications across the globe. Red Hat and Microsoft collaborate to offer several destination options to run JBoss EAP on Azure. The options include Azure Virtual Machines/VM Scale Sets, Azure Red Hat OpenShift (ARO), and App Service. The options are linked in the Learn more section below.

You can reach out to the engineering team developing these offers by clicking the CONTACT ME button. Program managers, architects and engineers will get in touch and can assist you for free with your Azure migration.

JBoss EAP on VMs/VMSS

These solutions automate most boilerplate steps to provision JBoss EAP on Azure VMs/VMSS. The use cases covered include standalone and clustered deployments. The solutions can also automate load-balancing using Azure App Gateway as well as database (Oracle DB/Azure PostgreSQL/Azure SQL/Azure MySQL) integration. Both Bring-Your-Own-Subscription (BYOS) or Pay-as-You-Go (PAYGO) is supported. Once initial provisioning is complete, you are completely free to customize deployments further. The overall solution also includes JBoss EAP VM base images as well step-by-step provisioning instructions without automation.

JBoss EAP on ARO

This solution automates most boilerplate steps to provision JBoss EAP on ARO. The automatically provisioned resources can include an ARO cluster, the JBoss EAP Operator, and an application using Source-To-Image (S2I). Once initial provisioning is complete, you are completely free to customize deployments further. The solution supports a Bring-Your-Own-Subscription (BYOS) model. The overall solution also includes step-by-step provisioning instructions on ARO without automation.

JBoss EAP on App Service

JBoss EAP on Azure App Service is a fully managed solution with built-in infrastructure management, security, patching, monitoring, and scaling. The solution is based on a Pay-as-You-Go (PAYGO) model, with integrated commercial support for both JBoss EAP and App Service.

Name

JBoss EAP standalone on RHEL VM

Search results summary

Run JBoss EAP workloads on Azure Virtual Machines and VM Scale Sets

Short description

JBoss EAP is a leading Java application server running some of the most mission-critical enterprise applications in the world. This offer accelerates deploying JBoss EAP workloads to Azure Virtual Machines and VM Scale Sets.

Description

This offer provisions a single Red Hat JBoss EAP server with JBoss EAP management console. All supporting Azure resources are automatically provisioned.

  • Computing resources
    • A RHEL 8.6 VM with the following settings based on user's choice:
      • Choice of VM size
  • Network resources
    • A virtual network and a subnet if users choose to create a new virtual network. You can also choose to deploy into a pre-existing virtual network.
    • A network security group if users choose to create a new virtual network.
    • A network interface with a private IP address.
  • Key software components
    • A JBoss EAP 7.4 standalone instance with user provided admin credentials. The EAP_HOME is /opt/rh/eap7/root/usr/share/wildfly.
    • OpenJDK 8, 11, or 17. The JAVA_HOME is a subdirectory of /usr/lib/jvm, depending on the selected JDK version.
    • This JBoss EAP host can be registered to an existing Red Hat Satellite server for management.

Red Hat JBoss Enterprise Application Platform (JBoss EAP) is a leading Java application server running some of the most mission-critical enterprise applications in the world.

This offer automatically provisions several Azure resources to quickly create a JBoss EAP deployment on Azure Virtual Machines or VM Scale Sets. The automatically provisioned resources include virtual network, storage, network security group, OpenJDK, Red Hat Enterprise Linux (RHEL), and JBoss EAP. The offer supports the latest version of RHEL 8, a choice of OpenJDK 8, 11, or 17, and JBoss EAP 7

You can provision different types of JBoss EAP deployments:

  • A single instance on a VM.
  • A JBoss EAP cluster on a number of VMs (with or without domain mode enabled).
  • A dynamic JBoss EAP cluster on VM Scale Sets.

This offer requires a JBoss EAP subscription and works on a Bring-Your-Own-Subscription (BYOS) basis. For the RHEL part of the offer, you have a choice to use either Pay-As-You-Go (PAYGO) or BYOS. In case of PAYGO, there is an additional hourly RHEL subscription charge for using this offer on top of the normal Azure compute, network and storage costs. To use RHEL BYOS, you must contact Red Hat to get your subscription enabled on Azure. Once you do so, the RHEL BYOS options will become visible as plans.
